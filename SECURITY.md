# Security Policy

This project is an open-source front-end application and may interact with authentication, upload, and user profile APIs.

## Reporting a Vulnerability

Please do not open a public issue for sensitive security reports.

Until a dedicated security contact is published, report vulnerabilities through a private maintainer channel.

Useful report details include:

- Affected page, API call, or workflow
- Steps to reproduce
- Impact and affected data
- Suggested fix, if known

## Sensitive Data

Do not commit:

- Real API keys or tokens
- App secrets
- Production environment files
- Private callback URLs
- User data exports
- Server credentials
