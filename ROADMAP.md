# Roadmap

This roadmap focuses on making the front-end project stable, understandable, and contributor-friendly before expanding the wider product ecosystem.

## Phase 1: Open Source Baseline

- [x] Prepare H5 development and build scripts
- [x] Document environment variables
- [x] Add license and contribution guide
- [x] Add repository hygiene rules through `.gitignore`
- [x] Clarify backend dependency in README
- [ ] Add screenshots or a short demo recording
- [ ] Publish a public demo environment or mock API

## Phase 2: Core Experience

- [x] Plant list and search
- [x] Plant detail page
- [x] Plant publishing flow
- [x] Community diary flow
- [x] Login, guest login, development login, and callback handling
- [ ] Improve empty, loading, and error states
- [ ] Strengthen form validation and publish feedback
- [ ] Improve upload retry and failure handling
- [ ] Improve expired-session recovery

- [ ] Add ESLint and Prettier or another shared code style setup
- [ ] Add basic unit tests
